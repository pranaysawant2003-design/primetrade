# Submission Package

Files:
- `ASSESSMENT_REPORT.md` — executive report with methodology, findings, strategy recommendations, and limitations.
- `Trader_Sentiment_Assessment.xlsx` — polished dashboard and detailed analytical tables.
- `Trader_Sentiment_Analysis.ipynb` — notebook structure for a company submission.
- `analysis.py` — reproducible command-line analysis script.
- `results.json` and summary CSV files — machine-readable outputs.
- PNG charts — presentation-ready visualizations.

Recommended submission:
1. Submit the notebook or `analysis.py` as the code deliverable.
2. Submit the Excel workbook as the business-facing analysis.
3. Submit `ASSESSMENT_REPORT.md` as the written interpretation.

The code expects the two company datasets to be supplied separately. Rename them to:
- `historical_trader_data.csv`
- `bitcoin_sentiment.csv`

or change the paths in the notebook / command line.
